NAME


pod2usage - print usage messages from embedded pod docs in files 
SYNOPSIS






\fBpod2usage\fR 12 



[-help] [-man] [-exit exitval] [-output outfile] [-verbose level] [-pathlist dirlist] [-formatter module] file 
OPTIONS AND ARGUMENTS






\fB-help\fR 8 



Print a brief help message and exit. 

\fB-man\fR 8 



Print this command's manual page and exit. 

\fB-exit\fR \fIexitval\fR 8 



The exit status value to return. 

\fB-output\fR \fIoutfile\fR 8 



The output file to print to. If the special names *(L"-*(R" or *(L">&1*(R" or *(L">&STDOUT*(R" are used then standard output is used. If *(L">&2*(R" or *(L">&STDERR*(R" is used then standard error is used. 

\fB-verbose\fR \fIlevel\fR 8 



The desired level of verbosity to use: 



    1 : print SYNOPSIS only     2 : print SYNOPSIS sections and any OPTIONS/ARGUMENTS sections     3 : print the entire manpage (similar to running pod2text) 



\fB-pathlist\fR \fIdirlist\fR 8 



Specifies one or more directories to search for the input file if it was not supplied with an absolute path. Each directory path in the given list should be separated by a ':' on Unix (';' on MSWin32 and s-1DOSs0). 

\fB-formatter\fR \fImodule\fR 8 



Which text formatter to use. Default is Pod::Text, or for very old Perl versions Pod::PlainText. An alternative would be e.g.  Pod::Text::Termcap. 

\fIfile\fR 8 



The pathname of a file containing pod documentation to be output in usage message format (defaults to standard input). 
DESCRIPTION




pod2usage will read the given input file looking for pod documentation and will print the corresponding usage message. If no input file is specified then standard input is read. 

pod2usage invokes the f(BIpod2usage() function in the Pod::Usage module. Please see *(L"pod2usage()*(R" in Pod::Usage. 
SEE ALSO




Pod::Usage, pod2text|(1)
