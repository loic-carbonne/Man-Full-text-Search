NAME

prlimit - get and set process resource limits

SYNOPSIS

prlimit [options]

prlimit [options]

command [ argument ...]

DESCRIPTION

Given a process id and one or more resources, prlimit tries to retrieve and/or modify the limits.

When command is given, prlimit will run this command with the given arguments.

The limits parameter is composed of a soft and a hard value, separated by a colon (:), in order to modify the existing values. If no limits are given, prlimit will display the current values. If one of the values is not given, then the existing one will be used. To specify the unlimited or infinity limit (RLIM_INFINITY), the -1 or 'unlimited' string can be passed.

Because of the nature of limits, the soft limit must be lower or equal to the high limit (also called the ceiling). To see all available resource limits, refer to the RESOURCE OPTIONS section.

\fIsoft\fP:\fIhard\fP Specify both limits.

\fIsoft\fP: Specify only the soft limit.

:\fIhard\fP Specify only the hard limit.

\fIvalue\fP Specify both limits to the same value.

GENERAL OPTIONS

\fB-h, --help\fP

Display help text and exit.

\fB--noheadings\fP

Do not print a header line.

\fB-o, --output \fIlist\fP

Define the output columns to use. If no output arrangement is specified, then a default set is used. Use --help to get a list of all supported columns.

\fB-p, --pid\fP

Specify the process id; if none is given, the running process will be used.

\fB--raw\fP

Use the raw output format.

\fB--verbose\fP

Verbose mode.

\fB-V, --version\fP

Display version information and exit.

RESOURCE OPTIONS

\fB-c, --core\fP[=\fIlimits\fR]

Maximum size of a core file.

\fB-d, --data\fP[=\fIlimits\fR]

Maximum data size.

\fB-e, --nice\fP[=\fIlimits\fR]

Maximum nice priority allowed to raise.

\fB-f, --fsize\fP[=\fIlimits\fR]

Maximum file size.

\fB-i, --sigpending\fP[=\fIlimits\fR]

Maximum number of pending signals.

\fB-l, --memlock\fP[=\fIlimits\fR]

Maximum locked-in-memory address space.

\fB-m, --rss\fP[=\fIlimits\fR]

Maximum Resident Set Size (RSS).

\fB-n, --nofile\fP[=\fIlimits\fR]

Maximum number of open files.

\fB-q, --msgqueue\fP[=\fIlimits\fR]

Maximum number of bytes in POSIX message queues.

\fB-r, --rtprio\fP[=\fIlimits\fR]

Maximum real-time priority.

\fB-s, --stack\fP[=\fIlimits\fR]

Maximum size of the stack.

\fB-t, --cpu\fP[=\fIlimits\fR]

CPU time, in seconds.

\fB-u, --nproc\fP[=\fIlimits\fR]

Maximum number of processes.

\fB-v, --as\fP[=\fIlimits\fR]

Address space limit.

\fB-x, --locks\fP[=\fIlimits\fR]

Maximum number of file locks held.

\fB-y, --rttime\fP[=\fIlimits\fR]

Timeout for real-time tasks.

EXAMPLES

\fBprlimit --pid 13134\fP

Display limit values for all current resources.

\fBprlimit --pid 13134 --rss --nofile=1024:4095\fP

Display the limits of the RSS, and set the soft and hard limits for the number of open files to 1024 and 4095, respectively.

\fBprlimit --pid 13134 --nproc=512:\fP

Modify only the soft limit for the number of processes.

\fBprlimit --pid $$ --nproc=unlimited\fP

Set for the current process both the soft and ceiling values for the number of processes to umlimited.

\fBprlimit --cpu=10 sort -u hugefile\fP

Set both the soft and hard CPU time limit to ten seconds and run 'sort'.

SEE ALSO

prlimit (2), ulimit (1)

NOTES

The prlimit system call is supported since Linux 2.6.36, older kernels will break this program.
