NAME


size - list section sizes and total size. 
SYNOPSIS




size [-A|-B|--format=compatibility]      [--help]      [-d|-o|-x|--radix=number]      [--common]      [-t|--totals]      [--target=bfdname] [-V|--version]      [objfile...] 
DESCRIPTION




The s-1GNU s0size utility lists the section sizes---and the total size---for each of the object or archive files objfile in its argument list.  By default, one line of output is generated for each object file or each module in an archive. 

objfile... are the object files to be examined. If none are specified, the file f(CW*(C`a.out*(C' will be used. 
OPTIONS




The command line options have the following meanings: 

\fB-A\fR 4 







\fB-B\fR 4 





\fB--format=\fR\fIcompatibility\fR 4 





Using one of these options, you can choose whether the output from s-1GNU s0size resembles output from System V size (using -A, or --format=sysv), or Berkeley size (using -B, or --format=berkeley).  The default is the one-line format similar to Berkeley's. 

Here is an example of the Berkeley (default) format of output from size: 



        $ size --format=Berkeley ranlib size         text    data    bss     dec     hex     filename         294880  81920   11592   388392  5ed28   ranlib         294880  81920   11888   388688  5ee50   size 



This is the same data, but displayed closer to System V conventions: 



        $ size --format=SysV ranlib size         ranlib  :         section         size         addr         .text         294880         8192         .data          81920       303104         .bss           11592       385024         Total         388392 



        size  :         section         size         addr         .text         294880         8192         .data          81920       303104         .bss           11888       385024         Total         388688 



\fB--help\fR 4 



Show a summary of acceptable arguments and options. 

\fB-d\fR 4 







\fB-o\fR 4 





\fB-x\fR 4 





\fB--radix=\fR\fInumber\fR 4 





Using one of these options, you can control whether the size of each section is given in decimal (-d, or --radix=10); octal (-o, or --radix=8); or hexadecimal (-x, or --radix=16).  In --radix=number, only the three values (8, 10, 16) are supported.  The total size is always given in two radices; decimal and hexadecimal for -d or -x output, or octal and hexadecimal if you're using -o. 

\fB--common\fR 4 



Print total size of common symbols in each file.  When using Berkeley format these are included in the bss size. 

\fB-t\fR 4 







\fB--totals\fR 4 





Show totals of all objects listed (Berkeley format listing mode only). 

\fB--target=\fR\fIbfdname\fR 4 



Specify that the object-code format for objfile is bfdname.  This option may not be necessary; size can automatically recognize many formats. 

\fB-V\fR 4 







\fB--version\fR 4 





Display the version number of size. 

\fB@\fR\fIfile\fR 4 



Read command-line options from file.  The options read are inserted in place of the original @file option.  If file does not exist, or cannot be read, then the option will be treated literally, and not removed. 

Options in file are separated by whitespace.  A whitespace character may be included in an option by surrounding the entire option in either single or double quotes.  Any character (including a backslash) may be included by prefixing the character to be included with a backslash.  The file may itself contain additional @file options; any such options will be processed recursively.
