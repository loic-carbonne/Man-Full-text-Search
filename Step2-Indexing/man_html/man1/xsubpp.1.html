NAME


xsubpp - compiler to convert Perl XS code into C code 
SYNOPSIS




xsubpp [-v] [-except] [-s pattern] [-prototypes] [-noversioncheck] [-nolinenumbers] [-nooptimize] [-typemap typemap] [-output filename]... file.xs 
DESCRIPTION




This compiler is typically run by the makefiles created by ExtUtils::MakeMaker or by Module::Build or other Perl module build tools. 

xsubpp will compile s-1XSs0 code into C code by embedding the constructs necessary to let C functions manipulate Perl values and creates the glue necessary to let Perl access those functions.  The compiler uses typemaps to determine how to map C function parameters and variables to Perl values. 

The compiler will search for typemap files called typemap.  It will use the following search path to find default typemaps, with the rightmost typemap taking precedence. 



        ../../../typemap:../../typemap:../typemap:typemap 



It will also use a default typemap installed as f(CW*(C`ExtUtils::typemap*(C'. 
OPTIONS




Note that the f(CW*(C`XSOPT*(C' MakeMaker option may be used to add these options to any makefiles generated by MakeMaker. 

\fB-hiertype\fR 5 



Retains '::' in type names so that *(C+ hierarchical types can be mapped. 

\fB-except\fR 5 



Adds exception handling stubs to the C code. 

\fB-typemap typemap\fR 5 



Indicates that a user-supplied typemap should take precedence over the default typemaps.  This option may be used multiple times, with the last typemap having the highest precedence. 

\fB-output filename\fR 5 



Specifies the name of the output file to generate.  If no file is specified, output will be written to standard output. 

\fB-v\fR 5 



Prints the xsubpp version number to standard output, then exits. 

\fB-prototypes\fR 5 



By default xsubpp will not automatically generate prototype code for all xsubs. This flag will enable prototypes. 

\fB-noversioncheck\fR 5 



Disables the run time test that determines if the object file (derived from the f(CW*(C`.xs*(C' file) and the f(CW*(C`.pm*(C' files have the same version number. 

\fB-nolinenumbers\fR 5 



Prevents the inclusion of '#line' directives in the output. 

\fB-nooptimize\fR 5 



Disables certain optimizations.  The only optimization that is currently affected is the use of targets by the output C code (see perlguts). This may significantly slow down the generated code, but this is the way xsubpp of 5.005 and earlier operated. 

\fB-noinout\fR 5 



Disable recognition of f(CW*(C`IN*(C', f(CW*(C`OUT_LIST*(C' and f(CW*(C`INOUT_LIST*(C' declarations. 

\fB-noargtypes\fR 5 



Disable recognition of ANSI-like descriptions of function signature. 

\fB-\*(C+\fR 5 



Currently doesn't do anything at all.  This flag has been a no-op for many versions of perl, at least as far back as perl5.003_07.  It's allowed here for backwards compatibility. 

\fB-s=...\fR or \fB-strip=...\fR 5 



This option is obscure and discouraged. 

If specified, the given string will be stripped off from the beginning of the C function name in the generated s-1XSs0 functions (if it starts with that prefix). This only applies to XSUBs without f(CW*(C`CODE*(C' or f(CW*(C`PPCODE*(C' blocks. For example, the s-1XS:s0 



  void foo_bar(int i); 



when f(CW*(C`xsubpp*(C' is invoked with f(CW*(C`-s foo_*(C' will install a f(CW*(C`foo_bar*(C' function in Perl, but really call f(CWbar(i) in C. Most of the time, this is the opposite of what you want and failure modes are somewhat obscure, so please avoid this option where possible. 
ENVIRONMENT




No environment variables are used.
